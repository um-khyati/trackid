04 — Motion Principles

One easing curve. One philosophy: weight, not bounce.

The shared easing curve

export const EASE = [0.22, 1, 0.36, 1];
// Use this cubic-bezier array as the
// `ease` value in EVERY Framer Motion
// transition. Never use Framer's
// built-in "bounce" or "spring" with
// default values — they read as
// playful, not premium.

Standard fade-in-up (use everywhere)

const fadeUp = {
  initial: { opacity: 0, y: 24 },
  whileInView: { opacity: 1, y: 0 },
  viewport: { once: true, margin: "-80px" },
  transition: { duration: 0.6, ease: EASE }
};
// Import this ONE object from
// /src/motion/variants.js
// Do not write ad-hoc transitions
// per section.

Rule — All shared motion variants live in /src/motion/variants.js. Every section imports from there. If your section needs a motion pattern that doesn't exist yet, add it to that shared file with a clear name — don't write a one-off transition inline in your component. Always wrap motion in a check against useReducedMotion() — shown in the reference section below.

07 — Don'ts

The specific mistakes that cause six different "premiums."

✕ Using an arbitrary Tailwind value

bg-[#3a3a3a] · text-[15px] · p-[18px]

If you need it, it's missing from the config — ask, don't invent.

✕ Display font on body copy

font-display on a paragraph

Fraunces is for headlines and the wordmark only. Long-form Fraunces reads as a wedding invitation, not engineering.

✕ Stock alert colors

text-red-500 · bg-green-500

Use safe / alert — Tailwind's default red/green read as a cheap admin dashboard, not a luxury brand.

✕ A new divider style

<hr> · a plain border-t line

Every section boundary uses .divider-teardrop. One motif, repeated, is what makes it feel designed on purpose.

✕ Framer Motion spring/bounce defaults

type: "spring" with default stiffness

Always use the shared EASE cubic-bezier from /src/motion/variants.js. Springy motion reads as playful-app, not jewelry-grade.

✕ Skipping reduced-motion

an animation with no fallback check

Every motion-bearing component checks useReducedMotion() before animating — no exceptions, see Section 04 above.
